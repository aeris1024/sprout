## Description

## 背景

GUI常駐中に追跡ファイルの変更を検知できると、「未保存の変更あり」の表示や、一定条件での自動コミットができる。コミットし忘れて作業を上書きしてしまう事故を減らす。

## 実装方針

1. **変更検知**: Tauri側でファイル監視(Rustのnotifyクレート、またはtauri-plugin-fs-watch)を使い、開いているプロジェクトの追跡ファイル(`status --json`の`tracked`一覧)を監視する。イベントはデバウンス(例: 2秒)してまとめる。
2. **変更ありインジケータ**: 変更イベント後に`status --json`を再取得し、タイトルバーやツリービューのHEADノードに「未保存の変更あり」バッジを表示する。
3. **自動スナップショット(オプトイン)**: 設定で有効化した場合のみ、「最後の変更からN分静止したら自動コミット」を実行する。コミットメッセージは`auto: 2026-07-16 01:30`のような定型とし、自動コミットであることが履歴で区別できるようにする(ラベル機能TASK-28があれば`auto`ラベルを付ける)。
4. **安全対策**:
   - 自動コミットは`commit`の「nothing to commit」エラーを正常系として無視する。
   - 別の操作(restore等)の実行中はスキップする(ロックエラーを再試行せず次の静止を待つ)。
   - 監視は追跡ファイルの変更のみ対象とし、未追跡ファイルの追加では自動コミットしない。
5. **設定画面**: 有効/無効、静止時間(分)を設定できるようにし、既定は無効とする。

## 注意

エディタ系ソフトの保存は一時ファイル書き込み+リネームのことが多く、イベントが複数回飛ぶ。デバウンスと「静止時間」の設計でカバーする。自動コミットが増えると履歴が埋まるため、ツリービュー側で自動コミットを折りたたむ表示は将来課題としてよい。
